# refactor

プロジェクト全体の品質分析とリファクタリング実行

プロジェクト全体のコード品質を客観的に分析し、段階的リファクタリングを実行します。

## 実行フェーズ

### フェーズ1: 品質分析・提案
1. **コード構造分析**：
   - ファイル組織・モジュール責任分離
   - 依存関係の循環・複雑度
   - アーキテクチャパターンの妥当性

2. **技術的負債の特定**：
   - 重複コード（DRY原則違反）
   - 長すぎる関数・クラス
   - 複雑な条件分岐
   - エラーハンドリングの一貫性
   - @ts-nocheck等の型チェック回避

3. **型安全性・TypeScript活用度**：
   - any型の使用箇所
   - 型定義の一貫性
   - インターフェース設計
   - ジェネリクス活用度

4. **テストコード品質**：
   - テストカバレッジ・構造
   - 重複・冗長性
   - モック・スタブ使用状況

5. **パフォーマンス・最適化**：
   - 非効率な処理
   - メモリリーク可能性
   - 不要な再計算・再レンダリング

6. **コメント・ドキュメント**：
   - 日本語統一（プロジェクト方針）
   - JSDoc形式の一貫性
   - 自己説明的コード vs 説明コメント

### フェーズ2: ユーザー合意・計画策定
- 優先度別リファクタリング対象リスト作成
- 各対象の改善案・期待効果・影響範囲
- 実装難易度・リスク評価
- 段階的実装計画の提示
- **ユーザー合意取得（必須）**

### フェーズ3: 段階的実装（合意後）
1. **事前確認**：
   - 現在のテスト実行（npm test）
   - Git状態確認・バックアップ準備

2. **リファクタリング実行**：
   - 低リスクから段階的実施
   - 各段階でのテスト検証
   - エラー発生時の自動ロールバック

3. **品質確認**：
   - 全テスト実行・品質確認
   - lint・typecheck実行
   - パフォーマンス確認

4. **コミット作成**：
   - Conventional Commits準拠
   - Claude Code attribution付き

## 重要原則
- **テスト優先**: 実装前後で必ずテスト実行
- **段階的実施**: 小さな変更を積み重ね
- **透明性確保**: 全工程をユーザーに報告
- **安全性重視**: 問題時の即座ロールバック
- **品質保証**: テスト100%通過が完了条件

動作優先で実装されたコード・テストを最新要件で体系的見直し、持続可能な高品質コードベースに進化させます。
